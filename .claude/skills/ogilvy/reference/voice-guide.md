# Ogilvy Voice Guide

## Tone Dimensions

| Dimension | Position | Notes |
|-----------|----------|-------|
| Formal ↔ Casual | Slightly formal | Professional but not stiff. No jargon for jargon's sake. |
| Serious ↔ Playful | Serious with edge | We have opinions. We state them clearly. Wit is welcome; whimsy is not. |
| Humble ↔ Confident | Confident | We know what we built and why it matters. But confidence comes from proof, not assertion. |
| Technical ↔ Accessible | Context-dependent | Match the audience. DS leads get different depth than CTOs. |

## Register Examples

### Landing Page (Confident, Benefit-Led)

**Do:**
> Your design system was built for humans. Now AI is writing the code. One of them has to change. Preset makes sure it's not your standards.

**Don't:**
> Preset is a powerful, cutting-edge platform that revolutionizes how teams manage their design systems with AI-powered capabilities.

### Launch Post (Problem > Pain > Solution)

**Do:**
> AI coding tools are fast, cheap, and wrong. They generate code that looks right but isn't — wrong tokens, wrong patterns, wrong intent. Your team fixes it. Every. Single. Time.

**Don't:**
> We're excited to announce our new platform that helps teams work better with AI tools! It's been an incredible journey and we can't wait for you to try it.

### Tweet (Punchy, Contrarian)

**Do:**
> AI is your fastest, dumbest employee. Preset makes it literate.

**Don't:**
> Check out our amazing new design system platform! It's going to change the way you work with AI.

### Technical Blog (Authoritative, Detailed)

**Do:**
> When Claude generates a button component, it pulls from training data — not your design system. It doesn't know you use `--color-action-primary` instead of `bg-blue-500`. MCP changes this by providing real-time context.

**Don't:**
> AI tools are really bad at using design systems correctly, so we built something to fix that problem for developers everywhere.

### One-liner (Simple, Memorable)

**Do:**
> AI writes code fast. Preset makes it write code right.

**Don't:**
> Preset: The ultimate AI-powered design system management platform for modern teams.

## Words We Use

| Word | Why |
|------|-----|
| Executable | Design systems that run, not just document |
| Enforce | Active compliance, not passive hope |
| Context | What AI needs to do its job correctly |
| Drift | The gap between design intent and code reality |
| Intent | What the designer/developer actually meant |
| Compliance | Measurable adherence to design system rules |
| Preset | A complete, executable component specification |
| Surface | Where a design decision manifests |

## Words We Avoid

| Word | Why | Use Instead |
|------|-----|-------------|
| Revolutionary | Unearned | "New approach" or just describe it |
| Game-changing | Empty | Show the change, don't label it |
| Cutting-edge | Vague | Be specific about what's new |
| Best-in-class | Unproven | Show comparison data |
| Seamless | Nothing is | "Integrates with" + specifics |
| Powerful | Means nothing | Describe the specific capability |
| Leverage | Corporate speak | "Use" |
| Utilize | See above | "Use" |
| Synergy | Always | Never |
| Unlock | Overused | "Enable" or just describe the outcome |
| Empower | Vague | Describe the specific capability given |
| Solution | When used alone | Name the actual thing |
| Excited | Irrelevant to the reader | Focus on their benefit |
| Proud | Self-congratulatory | Focus on the outcome |
| Journey | Cringe | "Process" or "work" |

## Proof Patterns

Ogilvy always backs claims with one of these proof types:

### Metric Proof
> "Teams using Preset report 50% fewer design system violations in AI-generated code."

Requires: actual measurement, specific number, clear attribution.

### Mechanism Proof
> "Preset delivers your design tokens to AI tools via MCP in real time — so the AI uses `--color-action-primary` instead of guessing `bg-blue-500`."

Requires: clear explanation of how it works, specific example.

### Contrast Proof
> "Without Preset: AI generates a button with `bg-blue-500 text-white rounded-md`. With Preset: AI generates a button with your exact token values, spacing system, and component patterns."

Requires: concrete before/after, recognizable scenario.

### Authority Proof
> "Built on the Model Context Protocol (MCP) — the same standard used by Claude, Cursor, and VS Code."

Requires: real association, accurate claim.

### Absence Proof
> "No other design system tool gives AI tools real-time access to your tokens and patterns."

Requires: verified competitive gap. Use carefully — markets move fast.

## Capitalization Rules

- **Preset** — Always capitalized (product name)
- **MCP** — Always uppercase (protocol acronym)
- **design system** — Lowercase (common noun)
- **AI** — Always uppercase
- Headlines — Sentence case, not Title Case
- Buttons — Sentence case
- No ALL CAPS for emphasis — use structure and word choice instead

## Punctuation

- Em dashes for emphasis and asides — like this
- Periods in body copy. Short sentences. Rhythm matters.
- No exclamation marks in marketing copy. Ever. Confidence doesn't shout.
- Colons for lists and explanations: like this
- Serial comma: tokens, patterns, and constraints
