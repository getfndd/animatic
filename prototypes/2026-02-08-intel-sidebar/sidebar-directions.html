UX answer on snooze/summary: Currently the summary is computed from intelligenceData (unfiltered), so snoozing has zero effect on the summary text. All directions except E fix this — A uses counts that update, B/D use filtered company lists, C uses a live scorecard. The fix is a one-line change: compute from needsAttention instead of intelligenceData.

On status dots: All directions remove colored status dots from the feed. The queue cards already communicate severity. The feed's job is temporal context — "what happened when" — not "how bad is it." Removing dots reduces visual noise by ~40% in the sidebar.

Rams' recommendation: Direction D (Reactive Prose) or A (Dense Timeline). D preserves the Weftly voice while fixing the reactivity bug and cleaning noise. A is highest density but loses warmth. Avoid B (company grouping loses temporal signal) and C (scorecard duplicates the queue cards' signal bars).
