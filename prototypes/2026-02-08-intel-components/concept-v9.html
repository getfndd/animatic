V9 — Research-Informed

Applies competitive research: Carbon 3-of-4 rule (color + text + shape), Sentry-style inline temporal signals (segment bars promoted to primary visual), evidence-first text, and two-density signal encoding (problem signals loud, healthy signals recede).

Design principles

  • 1. Evidence headline tells you what and why — not a score
  • 2. Segment bars show temporal pattern — like Sentry sparklines but for cadence
  • 3. Problem signals: tinted bg + colored value + dot = 3 redundant encodings
  • 4. Healthy signals: muted dot + gray value = visually recede, scannable as "fine"
  • 5. Contextual actions suggest what to do next — like V4

What changed from V8

Segment bars promoted: Updates and Meetings now show 6-month activity bars (like Sentry sparklines) instead of dots. You see the pattern — declining = gaps growing, stalled = one bar then nothing. Problem bars are colored (red segments), healthy bars use neutral gray.

Two-density encoding: Problem signal columns get Tailwind 100-shade tinted background (red-100, amber-100, blue-100) + primary text. Healthy signals use gray text, no background, smaller visual weight. Your eye goes straight to the trouble spots.

Text labels replace dots: No dots anywhere. Each signal shows a value (4/8, 2, 1d, OK) and a word label (Declining, Stale, Stable, Avg). These carry meaning without color per Carbon's 3-of-4 rule.

Consistent with badge/tag pattern: 100-shade backgrounds with text-primary follows the established UI pattern for tinted surface elements (badges, tags, signal columns).
