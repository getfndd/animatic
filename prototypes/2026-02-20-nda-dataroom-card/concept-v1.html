Design Decisions

1.

Flat, content-forward — Matches EngagementBreakdownPanel pattern. No nested cards. Uses typography + spacing for hierarchy, consistent with the rest of the Relationship tab.

2.

Status dots + badges — Reuses the same status dot pattern from NDADetailPanel (line 177). Green = signed, blue = sent, gray = not started. Consistent vocabulary.

3.

4-column metadata grid — Matches Relationship Details grid (line 1811 of InvestorProfile). Consistent layout rhythm within the tab.

4.

Contextual quick actions — Pill buttons change based on state. NDA signed = share folders. NDA sent = mark signed / remind. No NDA = send NDA. Never shows irrelevant actions.

5.

Position — Between Engagement Score and Relationship Details. Engagement is about signal data; NDA/Data Room is about document access state; Relationship is about personal connection. Logical grouping.

6.

No empty state bloat — When no NDA exists, we show a minimal 2-column grid (status + data room) instead of 4 columns. Quick actions provide the path forward.
