Summary of Issues

Systematic Issues (affect multiple pages)

Fix

Heading size: text-base vs text-lg

Activity, Data Room, Deal Docs, Investor Updates, Share Management, Investor Profile all use text-base font-semibold. Spec requires text-lg font-semibold.

Affected: 6 pages, ~15 empty states

Fix

Body color: text-secondary vs text-tertiary

Spec says description text should be text-text-tertiary. Most pages use text-text-secondary instead.

Affected: Activity, Data Room, Deal Docs, Investor Updates, Investor Profile

Align

Button spacing: mt-6 vs mt-4

Spec says mt-4 for CTA button. Many pages use mt-6.

Decision: Should we update spec to mt-6? It looks better with breathing room.

Align

Icon color: text-secondary vs text-tertiary

Spec says icon should be text-text-tertiary. Some pages use text-text-secondary (slightly darker).

Affected: Activity, Data Room, Investor Profile, Share Management

Page-Specific Issues

Fix

Investors: Generic DataTable empty state

"No data available" with Database icon is generic. Should say "No investors yet" with Users icon, contextual description, and "Add Investor" CTA.

Fix

Pipeline: Undersized column empty state

48px icon circle with 20px icon, text-xs text-disabled body. Should match spec sizing even if compact variant.

Fix

Data Room: Uses primary-action buttons

"Use Template" and "Upload Documents" use primary-action. Spec says empty state CTAs should always be secondary-action to preserve page hierarchy.

Review

Data Room: "AI will summarize..." copy

Per AI-assumed design philosophy, avoid mentioning AI as the mechanism. Rewrite to focus on outcome: "Your files will be summarized, categorized, and organized automatically."

Review

Contacts: font-medium vs font-semibold

Uses text-lg (correct) but font-medium instead of font-semibold. Minor visual weight difference.

Spec Update Candidates

Discuss

Should mt-6 replace mt-4 for button spacing?

Most implementations use mt-6. The extra 8px gives better visual breathing room between description and CTA.

Discuss

Allow dual CTAs for first-run onboarding states?

Data Room and Contacts both use two buttons. Pattern spec says single CTA. Exception for onboarding contexts?

Discuss

Should body text be text-secondary or text-tertiary?

Spec says text-tertiary. Implementations mostly use text-secondary. text-secondary is more readable, especially for guidance text. Consider updating spec.
