5. Implementation Notes

MilestoneBlock Extension

  • Attrs: icon, title, description, date
  • Icon: Phosphor icon name from preset list (no emoji)
  • Date: optional, string format
  • HTML: div with data-milestone-block
  • Email: bordered card with icon container (no left accent bar)
  • Pattern: same as MetricCard (atom, draggable, ReactNodeViewRenderer)

AttachmentBlock Extension

  • Attrs: fileName, fileSize, fileType, fileUrl
  • Types: pdf | doc | sheet | slide | other
  • Colors: blue/green/purple/amber/gray by type
  • HTML: div with data-attachment-block
  • Email: anchor wrapping table row with border
  • Compact: single row, not tall card

QuoteBlock Extension

  • Attrs: quote, authorName, authorTitle
  • Distinct: from Blockquote (callout) — this is testimonial
  • HTML: div with data-quote-block
  • Email: bordered card, #fafafa bg, large quote mark (no left accent bar)
  • Attribution: em-dash prefix, name bold
  • Avatar: optional placeholder (omitted in email)

Common Patterns Across All Three

  • Node type: atom (not editable content), draggable: true
  • Renderer: ReactNodeViewRenderer (same as MetricCard)
  • Input style: invisible borders, visible on hover/focus
  • Top bar: block type label + icon (like AskBlock variant bar)
  • Card style: border-border-default rounded-md bg-surface-secondary/50
  • Email output: all inline styles, table-based fallbacks
  • Slash command: added to COMMANDS array in SlashCommands.jsx
  • Font size: 13px base for compose content areas
