# Ogilvy Messaging Framework

## Core Narrative

Design systems were built for humans. Now AI writes the code. The gap between design intent and generated output grows with every commit. Teams built careful systems of tokens, patterns, and constraints — and AI ignores all of it.

Preset bridges the gap. It gives AI tools real-time access to your design system via MCP, so generated code respects your tokens, follows your patterns, and enforces your constraints. Your design system becomes executable context, not ignored documentation.

## Value Propositions by Audience

### Design System Leads

**Primary pain:** AI-generated code drifts from the design system, creating cleanup work and eroding trust in the system.

**Value prop:** Preset makes your design system the source of truth for AI, not just humans. Every AI-generated component respects your tokens, patterns, and constraints automatically.

**Proof angles:**
- Token usage compliance metrics
- Drift reduction before/after
- Time saved on code review

### Frontend Developers

**Primary pain:** Fixing AI output to match design system conventions. Copy-pasting tokens. Looking up pattern docs.

**Value prop:** Preset feeds your design system directly to your AI tools. Stop fixing generated code — get it right the first time.

**Proof angles:**
- Reduced review cycles
- Faster component scaffolding
- Less context-switching to docs

### Design Ops / Engineering Managers

**Primary pain:** Governance at scale. Ensuring consistency across teams, repos, and AI tools.

**Value prop:** Preset enforces design system compliance at the point of creation — before code is even committed. Governance that works with your workflow, not against it.

**Proof angles:**
- Compliance rates across teams
- Reduced design system violations in PRs
- Faster onboarding for new developers

## Headline Formulas

### Formula 1: Contrast (Old Way vs New Way)

```
[Old painful reality]. [New possibility with Preset].
```

**Examples:**
- "AI writes code fast. Preset makes it write code right."
- "Your design system has rules. Now your AI tools follow them."
- "Documentation gets ignored. Executable presets don't."

### Formula 2: Problem Reframe

```
[Common assumption]. [Why it's wrong]. [What to do instead].
```

**Examples:**
- "You don't have an AI quality problem. You have an AI context problem. Give it your design system."
- "Design systems aren't broken. They're just invisible to AI. Preset fixes that."

### Formula 3: Punchy Contrast

```
[Short provocative statement].
```

**Examples:**
- "AI is your fastest, dumbest employee. Preset makes it literate."
- "Your AI tools can't read your Figma files. But they can read Preset."
- "Every AI-generated component is a design system violation waiting to happen."

### Formula 4: Benefit-Led

```
[Desirable outcome] + [mechanism].
```

**Examples:**
- "Design system compliance, enforced at the speed of AI."
- "Your tokens, patterns, and constraints — delivered to AI in real time via MCP."
- "Stop reviewing AI code for design system violations. Start shipping it."

## Landing Page Structure

### Hero Section
1. **Headline** — Benefit-led, one line (Formula 1 or 4)
2. **Subhead** — Problem reframe, 1-2 sentences
3. **CTA** — Specific action ("Start with your design system" not "Get started")
4. **Social proof** — Logo bar or specific metric

### Problem Section
1. **Section head** — Name the pain specifically
2. **Three pain points** — Concrete, recognizable scenarios
3. **Transition** — "There's a better way" or equivalent

### Solution Section
1. **Section head** — Benefit-first framing
2. **Three capabilities** — Each with icon, title, 1-sentence description
3. **Visual** — Product screenshot or demo showing the capability

### Proof Section
1. **Metrics** — Specific numbers (compliance %, time saved, drift reduced)
2. **Testimonials** — Real quotes from real users (when available)
3. **Case study teaser** — Brief outcome story

### CTA Section
1. **Headline** — Restate the core value prop differently
2. **CTA** — Same specific action as hero
3. **Objection handler** — "Free to start" or "No credit card required"
